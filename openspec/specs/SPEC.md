# SPEC: nx-devkit zero-config inference plugins

## Purpose

This repository extends Nx with a minimal, **config-file-inference-only** plugin family. Each plugin inspects a single config file already present in a TypeScript project and auto-injects the matching Nx target via `createNodesV2`. Consumers do not write `project.json`; targets are derived from files they already keep (`tsdown.config.ts`, `tsconfig.json`, `.oxlintrc.json`, `biome.json`, `vitest.config.ts`).

The repo is a fork of [ThePlenkov/nx.ts](https://github.com/ThePlenkov/nx.ts), which itself forks [nx-devkit/nx.ts](https://github.com/nx-devkit/nx.ts). The two-PR strategy below opens concurrent PRs to both forks.

## Architecture

Four workspace packages under `packages/`:

| Package | Scope | Role |
|---|---|---|
| `packages/tsdown` | `@nx-devkit/tsdown` | Tool plugin. Owns `build` target inference. |
| `packages/oxlint` | `@nx-devkit/oxlint` | Tool plugin. Owns `lint` target inference. |
| `packages/biome` | `@nx-devkit/biome` | Tool plugin. Owns `format`, `format-check`, `lint` inference. |
| `packages/typescript-preset` | `@nx-devkit/typescript` | Preset plugin. Owns `typecheck`, `test`, `test:watch`, `test:coverage`. Exports reusable helpers. |

Per-tool plugins (tsdown, oxlint, biome) consume the preset via workspace dependency so they do not duplicate typecheck/test inference.

Tool plugins emit `nx:run-commands` executors (no custom executor packages), wrapped with `cache: true` and `dependsOn: ['^build']` where appropriate.

## File-trigger to target inference matrix

| File trigger | Plugin | Inferred target | Executor | Cache | Notes |
|---|---|---|---|---|---|
| `**/tsdown.config.ts` | `@nx-devkit/tsdown` | `build` | `nx:run-commands` (`npx tsdown`) | true | outputs `{projectRoot}/dist`; dependsOn `^build` |
| `**/tsconfig.json` | `@nx-devkit/typescript` | `typecheck` | `nx:run-commands` (`npx tsgo --build ${configFile}`) | true | `tsgo: boolean` option; inputs include `tsconfig.base.json` |
| `**/vitest.config.{ts,js,mts,mjs,cts,cjs}` | `@nx-devkit/typescript` | `test`, `test:watch`, `test:coverage` | `nx:run-commands` (`npx vitest run`) | true (test) / false (watch) | `test` dependsOn `^build` |
| `**/.oxlintrc.{json,yaml,yml,js,mjs,cjs}` | `@nx-devkit/oxlint` | `lint` | `nx:run-commands` (`npx oxlint .`) | true | inputs include `.oxlintrc.*`, `src/**/*` |
| `**/biome.json`, `biome.jsonc` | `@nx-devkit/biome` | `format`, `format-check`, `lint` | `nx:run-commands` (`npx biome ...`) | false (format) / true (check, lint) | format has side effects |

Workspace root is skipped by every plugin.

## Demo workspace plan

`apps/demo/` is a minimal Nx workspace that wires all five plugins via its own `nx.json`. It contains three example projects under `apps/demo/projects/`:

1. **library** - `tsdown.config.ts` + `tsconfig.json` + `.oxlintrc.json` + `biome.json` + `vitest.config.ts` + `src/index.ts` + `tests/index.spec.ts`.
2. **app** - same config set plus a `src/main.ts` entry.
3. **monorepo-pkg** - same config set as a workspace-pkg-style package (publishable).

`scripts/e2e.sh` runs `cd apps/demo && bunx nx run-many -t build typecheck lint format-check test` and asserts exit 0. The demo serves as living documentation and the e2e gate.

## Skills catalog

`skills/` ships six installable skills for [skills.sh](https://skills.sh):

| Skill | Purpose |
|---|---|
| `nx-devkit-tsdown` | Install + targets generated by `@nx-devkit/tsdown`. |
| `nx-devkit-oxlint` | Install + targets generated by `@nx-devkit/oxlint`. |
| `nx-devkit-biome` | Install + targets generated by `@nx-devkit/biome`. |
| `nx-devkit-typescript` | Install + preset rationale + shared typecheck/test targets. |
| `nx-vs-nx` | Positioning: we EXTEND Nx with config-file inference, we do NOT replace it. Comparison table vs `@nx/js`, `@nx/vite`, `@nx/eslint` plugins. |
| `nx-e2e-demo` | Clone → `bun install` → run `apps/demo`. |

Each `SKILL.md` has YAML frontmatter (`name`, `description`) per the skills.sh spec and stays under 500 lines.

## TDD workflow

1. Write a failing `*.spec.ts` alongside `plugin.ts` using `vitest` + `@nx/devkit` testing helpers + `memfs`.
2. Run `bun test` to confirm RED.
3. Implement minimum to pass.
4. Refactor.
5. Commit + push.
6. OpenSpec `validate` must pass before merge.

Every plugin bead follows this order: tests first, implementation second, README third.

## Two-PR strategy

Two open UNMERGED pull requests are opened from the same convoy feature branch `convoy/nx-ts-5-plugins-demo-skills-2-prs-opensp/e06d06b1/head`:

1. **Fork PR** → [ThePlenkov/nx.ts](https://github.com/ThePlenkov/nx.ts) (origin). Targets `main`.
2. **Upstream PR** → [nx-devkit/nx.ts](https://github.com/nx-devkit/nx.ts). Targets `main`.

The branch is pushed to both remotes; both PRs reference `openspec/specs/SPEC.md` and the bead convoy. No merges performed by the polecat. Refinery reviews.

## Quality gates

- `bun install` succeeds.
- `bun run lint` (biome) clean.
- `bun run format:check` clean.
- `bun test` green.
- `bun run spec:check` no duplicate target/inference definitions.
- `bunx openspec validate` no errors.
- `bash scripts/e2e.sh` exit 0.
